================================================================================
MODEL EXPLANATION
================================================================================

HOW IT WORKS:
--------------------------------------------------------------------------------
1. The model learns patterns from teams' offensive and defensive statistics
2. It combines performance metrics like shooting efficiency, rebounds, assists
3. Historical performance indicators (previous season win %) are considered
4. The model predicts the conference rank directly as a numeric value
5. Predictions are evaluated against actual final standings

DATA LEAKAGE PREVENTION:
--------------------------------------------------------------------------------
The following variables were EXCLUDED to prevent data leakage:
- rank, won, lost (outcome variables)
- playoff, firstRound, semis, finals (derived from rank)
- rs_win_pct, po_W, po_L, po_win_pct (direct win/loss metrics)
- homeW, homeL, awayW, awayL, confW, confL (split win/loss records)

FEATURES USED:
--------------------------------------------------------------------------------
The model uses:
- Offensive statistics (field goals, free throws, 3-pointers, rebounds, etc.)
- Defensive statistics (opponent's shooting, turnovers, steals, blocks)
- Aggregate metrics (pythag win %, team strength, expected roster performance)
- Historical indicators (previous season win %, win % change)

FUTURE IMPROVEMENTS:
--------------------------------------------------------------------------------
1. Incorporate player performance metrics (scoring, efficiency, experience)
2. Add coach performance indicators (years of experience, past success)
3. Include rookie development features
4. Consider team chemistry and roster stability metrics
5. Experiment with ensemble methods or deep learning approaches
6. Add cross-validation for more robust evaluation
7. Feature engineering: create interaction terms and derived statistics
